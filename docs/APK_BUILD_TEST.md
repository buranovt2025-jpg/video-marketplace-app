## APK сборка и минимальный тест-план (Android)

### 1) Сборка APK (release)

На машине/сервере где установлен Flutter:

```bash
bash scripts/build_apk.sh
```

Полезные переменные окружения:
- `PROJECT_DIR` — путь к репозиторию
- `FLUTTER_BIN` — путь к `flutter`
- `BRANCH` — ветка (если нужно переключиться)

Артефакт обычно лежит:
- `build/app/outputs/flutter-apk/app-release.apk`

### 2) Установка на телефон

1) Передайте APK на телефон.
2) Разрешите установку из неизвестных источников (один раз).
3) Установите APK.

### 3) Минимальный smoke test (10–15 минут)

#### Гость
- Открывается лента
- Переход в “Вход” работает
- “Заказы” показывает приглашение войти (без краша)

#### Buyer (buyer@demo.com / demo123)
- Логин
- Лента → открыть товар → “Купить” → корзина → checkout → создать заказ
- Заказы → открыть трекинг

#### Seller (seller@demo.com / demo123)
- Логин
- “Мои товары” открывается
- Создать товар (с картинкой URL) и сохранить
- Создать рилс/историю (пока через URL) и проверить отображение в ленте

#### Courier (courier@demo.com / demo123)
- Логин
- Экран курьера открывается, вкладки не падают
- Если есть заказ: открыть детали → “В пути” → “Доставлен”

#### Admin (admin@demo.com / admin123)
- Логин
- Dashboard/Users/Orders/Content открываются без ошибок

### 4) Что фиксировать в отчёте
- Версия/commit (по `.last_build_id` на сервере и commit в репозитории)
- Скрин/видео проблемы
- Роль (buyer/seller/courier/admin) + шаги воспроизведения
